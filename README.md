# SEILTANZER TERMINAL

Локальный веб-дашборд поддержки решений для дискреционного трейдера, торгующего
price-action сетапы (FVG / sweep / AMD) на индексах, металлах и валютах через
проп-фирмы. В каждый момент открытой сделки терминал показывает **вероятность
дохождения до тейка раньше стопа**, обогащённую данными опционного рынка, в
«бумажном» визуальном стиле квант-терминала.

> Это **инструмент поддержки решений, не сигнальный сервис**. Всё, что показано
> на экране, выводимо из реально полученных данных. Каждое число имеет tooltip
> «как посчитано» (формула + источник). Нет данных — панель показывает честное
> «нет данных», а не последнее/выдуманное значение.

---

## Запуск

```bash
pip install -e .
python -m seiltanzer            # терминал на http://127.0.0.1:8790
python -m seiltanzer --demo     # демо-режим: синтетический поток (бейдж DEMO)
```

Опции: `--host`, `--port 8790`, `--data-dir .` (куда класть `trades.db` /
`cache.db`).

Требуется Python 3.11+. Основные зависимости: FastAPI, uvicorn, numpy, pandas,
yfinance. Данные: `yfinance` — котировки и опционные цепочки ETF-прокси
(QQQ→NAS100, SPY→SP500, DIA→US30, GLD→XAU), индексы `^VIX` / `^GVZ` / `^V1X`.
Кэш пишется на диск в sqlite.

### Проверка честного «нет данных»

Выключите интернет (или запустите там, где Yahoo недоступен) — все панели
перейдут в состояние «нет данных» с текстом причины, а UI останется живым.
Демо-режим (`--demo`) синтезирует все потоки и помечает бейджем DEMO каждую
панель — для проверки анимаций без сети.

## Панели

1. **Шапка «аккаунт»** — конфигурация, активный сетап (№ и имя), фаза
   (1ph/2ph/funded), баланс % и строка динамической матрицы риска (риск %,
   целевой RR × ATR-множитель, режим). Справа: статусы фидов (цена / цепочка /
   VIX), UTC-часы, кнопки сделки.
2. **Probability Lattice** — живая доска Гальтона. Слева: P сейчас (герой с
   полосой [lo–hi]), r, EV холд / EV лестница, шариков упало, доля зелёных,
   «сходимость». Справа: доска с 8 рядами штырьков, наклон которых = текущая P
   модели; шарики падают с физикой отскоков, финальная корзина **сэмплируется
   заранее из МК-распределения** терминального R — доска красива и статистически
   честна.
3. **Strike Landscape** — гряда risk-neutral плотностей (Бриден–Литценбергер)
   из снапшотов реальных опционных цепочек. Вертикали: вход / стоп / тейк /
   GEX-уровни. Выноска: P(за тейк) и P(за стоп) по плотности рынка рядом с P
   модели — видно расхождение рынка и вашей статистики.
4. **Карта уровней** — коридор цены вокруг позиции: FVG-зоны (введённые вручную),
   GEX-уровни, implied move ±1σ до конца сессии, VWAP дня, вход/стоп/тейк.
5. **Фильтры и лестница фиксации** — чипы VIX / GVZ / DV1X / ATR-фаза /
   Теханализ (PASS / BLOCK / MANUAL) и 6 рубежей лестницы с БУ после 1.5R.
6. **Журнал** — таблица сделок из sqlite, экспорт CSV, автопересчёт статистики
   сетапов и формулы 2α/(α+β); при ≥20 сделок по сетапу калибровка переключается
   на журнал (бейдж «калибровка: журнал»).

## Математическое ядро (`seiltanzer/core/`)

**`prob.py` — вероятности и Монте-Карло**

- **First-passage.** Путь сделки в R-координатах: `dX = μ dt + σ dW`, стоп −1,
  тейк +T. Функция масштаба `s(x)=exp(−2μx/σ²)`;
  `P(тейк раньше стопа | x) = (s(x)−s(−1)) / (s(T)−s(−1))`
  (в коде — численно устойчивая форма через `expm1`).
- **Калибровка μ** бисекцией из условия `P(...|0) = winrate`. Winrate и T — из
  встроенной таблицы 16 сетапов либо из журнала (при ≥20 сделок, приоритет журнала).
- **Интервал неопределённости** — Уилсон 90% по винрейту → μ_lo/μ_hi → полоса
  [p_lo, p_hi]. Показывается всегда; при n<30 — единственно честное представление.
- **Монте-Карло** остатка сделки с поглощением на −1 и T (поправка броуновского
  моста устраняет смещение дискретизации): терминальное распределение,
  вероятности по горизонтам, EV «холд» vs EV «лестница фиксации».

**`options.py` — опционная математика**

- **Implied move** из ATM straddle ближайшей экспирации → `σ_implied` годовая.
- **Опционная поправка волы**: σ процесса масштабируется отношением
  `σ_implied / σ_baseline` (baseline — реализованная вола за 20 дней). В сжатом
  рынке далёкий тейк честно теряет вероятность на горизонте, в разогнанном —
  набирает. Если цепочки нет (DAX, FTSE, FX) — работаем без поправки и явно
  пишем это в UI.
- **Risk-neutral density** (Бриден–Литценбергер): `q(K) ≈ e^{rT} d²C/dK²`
  численно, сглаживание локальной квадратичной регрессией, обрезка отрицательных.
- **GEX-уровни** (помечены как эвристика): `GEX(K)=Σ γ(K)·OI·100·S²·0.01`,
  коллы +, путы −; zero-gamma flip и 3 крупнейших уровня. Допущение о
  позиционировании дилеров непроверяемо — только контекст.

**`risk.py` — риск-менеджмент (главы 2.1 / 2.6 / 2.9)**

- Динамическая матрица риска и целевого RR (формулы перенесены из
  Excel-калькулятора), поправка фазы проп-фирмы (1ph +2%, 2ph +1%).
- ATR-фаза `ratio = ATR(5)/ATR(20)`: шок (k=0.5), импульс (k=1.2), флэт (k=0.7),
  норма (k=1.0) — корректирует отображаемый целевой RR.
- Формула эффективности сетапа `2α/(α+β)` с интерпретацией.

## API

- `GET /api/state` — полное состояние (первый рендер): тик, гряда, журнал, сетапы.
- `WS /ws` — тики: цена, r, p, [p_lo, p_hi], фаза ATR, фильтры, МК, лестница, уровни.
- `GET /api/chain[?ticker=QQQ]` — гряда BL-плотностей + GEX + implied move (кэш ~10 мин).
- `POST /api/trade`, `POST /api/trade/close`, `POST /api/trade/zones` — жизненный
  цикл сделки.
- `GET /api/journal`, `POST /api/journal` (бэкфилл), `GET /api/journal.csv`.
- `GET /api/setups` — встроенная таблица 16 сетапов (winrate, n, RR) + журнальная
  статистика.
- `POST /api/account` — название / фаза / начальный капитал / баланс.

## Тесты

```bash
pytest -q
```

Покрыто ядро (`test_prob.py`, `test_risk.py`, `test_options.py`) и связка
журнал/движок/фильтры (`test_journal_engine.py`): калибровка и раунд-трип
first-passage, соответствие МК аналитике, влияние сжатия волы на горизонтную
вероятность, лестница с безубытком, восстановление σ и лог-нормальных хвостов из
синтетической цепочки, GEX-флип, матрица риска против контрольных точек Excel,
переключение калибровки на журнал.

## Стек

Backend: Python 3.11+, FastAPI + uvicorn, WebSocket для пуша тиков, sqlite для
кэша и журнала. Frontend: статический HTML / CSS / Canvas без фреймворков; все
анимации — canvas 60fps с деградацией до 30fps. Один процесс `python -m
seiltanzer` поднимает API и раздаёт фронт.

Дизайн-токены: `--paper #F4F2EC`, `--card #FFFFFF`, `--ink #14140F`,
`--rule #D8D5CC`, `--red #C6373C`, `--green #2E7D4F`, `--dim #8A877D`; шрифт IBM
Plex Mono; UPPERCASE-ярлыки 10px letter-spacing .12em; числа-герои tabular-nums.
