## Роль и цель

Ты собираешь **Seiltanzer Terminal** — локальный веб-дашборд для дискреционного трейдера,
торгующего price-action сетапы (FVG/sweep/AMD) на индексах, металлах и валютах через
проп-фирмы. Задача терминала: в каждый момент открытой сделки показывать **вероятность
дохождения до тейка раньше стопа**, обогащённую данными опционного рынка, в визуальном
стиле «бумажного» квант-терминала (описан ниже).

Это инструмент поддержки решений, НЕ сигнальный сервис. Всё, что он показывает,
должно быть выводимо из данных, которые он реально получил.

## Жёсткие запреты (нарушение = провал задачи)

- НИКАКИХ придуманных метрик: никакого Sharpe, «confidence 91%», total PnL,
  «predicted UP» без вычисления, которое можно проследить до входных данных.
- Каждое число на экране имеет tooltip «как посчитано» (формула + источник данных).
- Если данных нет (рынок закрыт, Yahoo не ответил, цепочка опционов пустая) —
  панель показывает состояние «нет данных», а не последнее/выдуманное значение.
- Вероятности с выборкой < 30 сделок всегда отображаются с интервалом
  неопределённости, никогда одним числом.

## Стек

- **Backend:** Python 3.11+, FastAPI + uvicorn, WebSocket для пуша тиков.
  Данные: `yfinance` (котировки + опционные цепочки QQQ/SPY/GLD, индексы ^VIX, ^GVZ если доступен, ^V1X/^GDAXI),
  polling 3–5 сек для цен, 5–10 мин для цепочек. Кэш на диск (sqlite или parquet).
- **Frontend:** статический HTML/JS/Canvas без фреймворков (или Vite + vanilla TS).
  Все анимации — canvas, 60fps, деградация до 30fps.
- **Хранилище журнала:** sqlite (`trades.db`): сделки пользователя
  (дата, сетап 1–16, инструмент, направление, вход, стоп, тейк, результат R, заметки).
- Один процесс: `python -m seiltanzer` поднимает API и раздаёт фронт на localhost:8790.

## Математическое ядро (модуль `core/prob.py`, покрыть тестами)

1. **Базовая модель первого достижения.** Путь сделки в R-координатах:
   `dX = mu*dt + sigma*dW`, стоп = −1, тейк = +T (целевой RR сетапа).
   Функция масштаба `s(x) = exp(-2*mu*x/sigma^2)`;
   `P(тейк раньше стопа | x) = (s(x)-s(-1)) / (s(T)-s(-1))`.
2. **Калибровка mu:** решить `P(...|0) = winrate` бисекцией. Winrate и T берутся
   из статистики выбранного сетапа (сначала из встроенной таблицы 16 сетапов,
   затем — из журнала пользователя, когда в нём ≥ 20 сделок по сетапу; приоритет журнала).
3. **Интервал неопределённости:** интервал Уилсона 90% по винрейту → mu_lo, mu_hi →
   полоса вероятности [p_lo, p_hi]. Показывается всегда.
4. **Опционная поправка волатильности (главная добавленная ценность):**
   - Из цепочки ближайшей экспирации QQQ (для NAS100) / SPY (SP500) / GLD (XAU)
     взять ATM straddle → implied move до экспирации → пересчитать в
     `sigma_implied` за оставшееся торговое время.
   - `sigma` процесса масштабировать отношением `sigma_implied / sigma_baseline`,
     где baseline — реализованная вола инструмента за 20 дней (из дневных баров Yahoo).
   - Эффект: в сжатом рынке далёкий тейк честно теряет вероятность, в разогнанном — набирает.
   - Если цепочки нет (DAX, FTSE, FX) — работаем без поправки и явно пишем это в UI.
5. **Risk-neutral density (Бриден–Литценбергер):** из mid-цен коллов ближайшей
   недельной экспирации: `q(K) ≈ e^{rT} * d²C/dK²` (численно, со сглаживанием
   сплайном и обрезкой отрицательных значений). Перевести страйки в R-координаты
   текущей сделки. Это данные для панели «Strike Landscape».
6. **GEX-уровни (эвристика, помечать как эвристику):** по open interest цепочки:
   `GEX(K) = Σ gamma(K)*OI*100*S²*0.01` с知знаком: коллы +, путы − (стандартное
   допущение «дилеры лонг коллы / шорт путы»). Найти zero-gamma flip и 3 крупнейших
   абсолютных уровня. Отобразить на карте уровней. В tooltip: «допущение о
   позиционировании дилеров не проверяемо, использовать как контекст».
7. **Фильтры стратегии (автоматизация главы 2.7/2.9 и сетапов):**
   - VIX>20 (сетапы 5, 6, 11), GVZ<18 (сетап 11), DV1X<19 (сетап 7; если тикер
     недоступен в Yahoo — статус «проверь вручную», не пропускать молча).
   - ATR-фильтр фазы: ratio = ATR(5)/ATR(20) на дневках инструмента;
     фазы: >1.5 шок (k=0.5), >1.15 импульс (k=1.2), <0.85 флэт (k=0.7), иначе норма (k=1.0).
     k корректирует отображаемый целевой RR, как в стратегии.
8. **Монте-Карло остатка сделки:** N путей из текущего r с калиброванными mu/sigma,
   поглощение на −1 и T; отдаёт: терминальное распределение, вероятность по горизонтам,
   EV «холд» vs EV «лестница фиксации» (10% на 1.0/1.25/1.5/1.75/2.0/2.2R,
   безубыток после 1.5R).

## Визуальная система (точное соответствие референсу)

Референс — «бумажный» терминал: светлый фон, плотная моноширинная типографика,
тонкие линейки, скупые красный/зелёный акценты.

Токены (CSS-переменные):
- `--paper: #F4F2EC` (фон), `--card: #FFFFFF`, `--ink: #14140F`,
  `--rule: #D8D5CC` (все границы 1px), `--red: #C6373C`, `--green: #2E7D4F`,
  `--dim: #8A877D`, `--accent-bg: #EEECE4`.
- Шрифты: заголовки и данные — IBM Plex Mono; ярлыки — IBM Plex Mono 10px,
  UPPERCASE, letter-spacing .12em. Числа-герои — 40–56px, tabular-nums.
- Карточки: белые, 1px граница `--rule`, радиус 3px, внутренние линейки-разделители.
- Пиктограммы статуса: точка + текст (`● LIVE`, `○ NO DATA`, `◐ DELAYED`).
- Никаких теней/градиентов, кроме лёгкой подложки под падающими шариками.

## Панели (сверху вниз)

### 1. Шапка «аккаунт»
Слева: название конфигурации, активный сетап (№ и имя), фаза счёта (1ph/2ph/funded),
баланс % и строка динамической матрицы риска (риск %, целевой RR, режим — из
встроенной таблицы главы 2.1). Справа: статусы фидов (цена / цепочка / VIX),
UTC-часы, кнопка «Новая сделка».

### 2. Probability Lattice — живая доска Гальтона (центральная панель)
- Слева колонка цифр: P сейчас (герой, с полосой [lo–hi]), r сейчас,
  EV холд / EV лестница, шариков упало, доля зелёных, «сходимость» =
  |доля зелёных − P модели| (показывает, что доска не врёт).
- Справа canvas: доска с 8 рядами штырьков. **Смещение каждого ряда = текущая
  P модели** (пересчитывается каждый тик). Каждые 400–700 мс падает шарик с
  физикой отскоков; финальная корзина определяется заранее сэмплом из
  МК-распределения терминального R, а траектория подгоняется — так доска
  красива И статистически честна. Корзины слева от нуля — красные, справа — зелёные,
  границы −1R и T подписаны. Гистограмма накапливается за сделку, сбрасывается
  кнопкой или при новой сделке.
- Подпись снизу: «наклон доски = вероятность из вашей статистики и опционной
  волы; шарики — исходы Монте-Карло, не рынок».

### 3. Strike Landscape — гряда распределений (реальные опционы)
- Ridgeline: 6–10 гряд. Верхние — risk-neutral density из цепочек за последние
  снапшоты (история интрадей, каждые 10 мин), нижняя жирная — актуальная.
- Ось X — цена, сверху вторая шкала в R текущей сделки. Вертикальные маркеры:
  вход (чёрный), стоп (красный), тейк (зелёный), GEX-уровни (пунктир, серый).
- Выноска на актуальной гряде: P(выше тейка) и P(ниже стопа) по risk-neutral
  плотности — рядом с модельной P, чтобы видеть расхождение рынка и вашей статистики.
- Если цепочек нет для инструмента — панель в состоянии «опционные данные
  недоступны для {инструмент}», без фейка.

### 4. Карта уровней (вместо фейкового графа связей)
Горизонтальная шкала цены вокруг позиции: FVG-зоны, введённые пользователем
вручную для сделки (верх/низ зоны, таймфрейм), fib 0.786 цели, GEX-уровни,
implied move (±1σ до конца сессии) как затенённый диапазон, VWAP дня.
Это рабочая замена «Relationship Graph»: показывает, в каком «коридоре» рынок
видит цену и где ваши уровни относительно него.

### 5. Лента фильтров и лестница фиксации
Слева: чипы фильтров (VIX / GVZ / DV1X / ATR-фаза / Теханализ>−30 — последний
вручную) — зелёный «PASS», красный «BLOCK», серый «MANUAL». Справа: 6 рубежей
лестницы с галочками пройденных и «стоп в БУ» после 1.5R.

### 6. Журнал (низ, сворачиваемый)
Таблица сделок из sqlite + кнопка экспорта CSV. Автопересчёт статистики сетапа
и формулы 2α/(α+β) из стратегии; при ≥20 сделок по сетапу — переключение
калибровки на журнал (с бейджем «калибровка: журнал»).

## API (минимум)

- `GET /api/state` — полное состояние (для первого рендера)
- `WS /ws` — тики: цена, r, p, [p_lo,p_hi], фаза ATR, фильтры
- `GET /api/chain?ticker=QQQ` — плотность BL + GEX + implied move (кэш 10 мин)
- `POST /api/trade` — открыть/закрыть сделку, `GET/POST /api/journal`
- `GET /api/setups` — встроенная таблица 16 сетапов (winrate, n, RR)
